Conciliador de pedidos x notas fiscais

Clique para subir a planilha de pedidos enviada ao fornecedor (.xlsx, .xls ou .csv)
Colunas reconhecidas: ID Escola, Coleção, Volume, Quantidade, Data do pedido, Projeto?, Tipo de faturamento. Essa base sempre acumula.
Por economia de leituras, essa lista não fica ao vivo — clique em "Atualizar" para buscar as mudanças mais recentes.
Clique para subir a base de Escola × CNPJ (.xlsx, .xls ou .csv)
Usada para achar o CNPJ da escola na hora de conferir a NF. Cada upload substitui a base inteira.
Clique para subir a base de-para (.xlsx, .xls ou .csv)
Colunas esperadas: Selo, Coleção, Volume, SKU. Cada upload substitui a base inteira. Se um SKU compõe mais de uma coleção, adicione uma linha para cada combinação.
Clique para subir a base de custos por SKU (.xlsx, .xls ou .csv)
Colunas esperadas: ID Escola, Coleção, Volume, SKU, Custo unitário. O custo pode variar por escola — cada upload substitui a base inteira.
Clique para subir a base de de-para de faturamento (.xlsx, .xls ou .csv)
Colunas esperadas: Tipo de faturamento na NF, Tipo de faturamento em Pedidos. Isso liga a frase longa que vem no XML (ex: "VENDA DE MERCADORIA...") ao termo curto que vocês usam em Pedidos (ex: "Venda"). Cada upload substitui a base inteira.
Clique para subir a planilha padrão de conciliação (dados do fornecedor no modelo padrão)
Colunas do modelo padrão: ID Escola, SKU, Quantidade, Nº do pedido, Data do pedido (a data em que nós fizemos o pedido, não a data em que o fornecedor respondeu). A ferramenta busca a combinação exata de Escola + SKU + Data do pedido, então não há ambiguidade entre pedidos parecidos. Se a quantidade informada for menor que a esperada pra esse SKU, o saldo restante fica guardado pra próxima conciliação completar.
